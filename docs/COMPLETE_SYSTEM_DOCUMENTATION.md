#### Why Mouse Sensor INSTEAD OF Camera?

| Feature | ESP32-CAM (2MP) | Mouse Sensor (ADNS-3050) | Winner |
| :-- | :-- | :-- | :-- |
| **Frame rate** | 25 fps | 6400 fps | Mouse (256Г— faster) |
| **Processing** | JPEG decode (40ms) | Raw pixels (0.3ms) | Mouse (133Г— faster) |
| **Cost** | вӮ№679 + вӮ№350 programmer | вӮ№0 (you have it) | Mouse (FREE!) |
| **Working distance** | 50-100mm | 2-12mm (adjustable) | Mouse (closer = controlled) |
| **Power draw** | 300mA | 50mA | Mouse (6Г— efficient) |
| **Interface** | Serial/WiFi (slow) | SPI (10MHz, fast) | Mouse |
| **Built-in LED** | No (need external) | Yes (but we replace) | Tie |
| **Motion tracking** | No | Yes (bonus feature!) | Mouse |

**Conclusion**: Mouse sensor is PERFECT for this application - fast, free, dual-purpose.

| Aspect | ESP32 (Software) | FPGA (Hardware) | Advantage |

| **SPI reading** | Software bit-bang or peripheral (sequential) | DMA hardware state machine (parallel) | FPGA (15Г—) |
| **Addition (900 values)** | Software loop (900 cycles) | Adder tree (10 cycles) | FPGA (90Г—) |
| **RGBвҶ’HSV** | Floating-point (slow) | Fixed-point pipeline (fast) | FPGA (15Г—) |
| **Classification** | If-else chain (sequential) | Parallel comparators (simultaneous) | FPGA (7Г—) |
| **Power efficiency** | 240MHz Г— 2 cores = 480M instructions/s @ 500mW | 100MHz Г— 1800 LUTs @ 200mW | FPGA (6Г— efficient) |
| **Determinism** | Interrupted by OS, WiFi, etc. (jitter) | Pure hardware (cycle-accurate timing) | FPGA (critical!) |
